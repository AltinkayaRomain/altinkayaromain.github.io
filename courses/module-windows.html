1. DNS (Domain Name System)

Sans DNS, pas d'Active Directory. Il transforme les noms d'hôtes en adresses IP (et inversement).

Concepts à maîtriser :

  • Zones de recherche directe : Nom → IP.
  • Zones de recherche inversée : IP → Nom (Enregistrements PTR).
  • Types d'enregistrements : A (IPv4), AAAA (IPv6), CNAME (Alias), MX (Mail), SRV (Services AD).
  • Zones intégrées à l'AD : Réplication sécurisée avec les autres DC.
PowerShell - Test de résolution DNS
Resolve-DnsName -Name dc01.formation.local

2. ADDS & Maîtres d'opérations (FSMO)

L'Active Directory n'est pas totalement multi-maître ; certains rôles sont uniques dans la forêt ou le domaine.

Les 5 rôles FSMO :

Niveau Forêt : 1. Contrôleur de schéma | 2. Maître de nommage de domaine.

Niveau Domaine : 3. PDC Émulateur (Temps/Mots de passe) | 4. Maître RID (ID uniques) | 5. Maître d'infrastructure.

3. Stratégies de Groupe (GPO)

Permet la configuration centralisée des postes et utilisateurs.

  • Ordre d'application (LSDOU) : Local → Site → Domaine → Unité d'Organisation (OU). La dernière appliquée gagne.
  • Exceptions : "Appliquer" (Force la GPO) et "Bloquer l'héritage".
  • Loopback Processing : Applique des paramètres utilisateur en fonction de l'ordinateur sur lequel il se connecte (Utile pour le RDS).

4. Architecture RDS (Remote Desktop Services)

Pour l'examen, sachez différencier les rôles :

  • Broker (Agent de connexion) : Répartit les sessions sur les serveurs.
  • Passerelle (Gateway) : Permet l'accès sécurisé depuis Internet via HTTPS (443).
  • Hôte de session : Le serveur qui exécute réellement les applications.

5. Messagerie & Flux SMTP (Exchange)

Exchange Server s'appuie sur l'AD pour sa base d'utilisateurs.

  • Connecteurs : Réception (Inbound) et Envoi (Outbound).
  • Protocoles : SMTP (25/587) pour le transport, IMAP (143/993) pour la consultation.
  • DAG (Database Availability Group) : Haute disponibilité des bases de données mails.

6. Industrialisation : WDS, MDT & WSUS

Comment gérer un parc de 500 machines sans les toucher une par une.

  • WDS : Envoie l'image système via le réseau (PXE).
  • MDT : Personnalise l'installation (Applications, drivers, scripts).
  • WSUS : Serveur de mise à jour local pour économiser la bande passante et valider les patchs avant déploiement.
