2. Switching Avancé (L2)

A. VLANs & Tagging (802.1Q)

Un VLAN découpe un switch physique en plusieurs switchs logiques.
Problème : Comment faire passer plusieurs VLANs dans un seul câble entre deux switchs ?
Solution : Le Trunk (802.1Q). Il ajoute une "étiquette" (Tag) de 4 octets dans l'en-tête de la trame Ethernet pour dire "Je viens du VLAN 10".

Native VLAN : C'est le seul VLAN qui passe dans le Trunk SANS étiquette. Par sécurité, changez-le (ne laissez pas le VLAN 1 par défaut) pour éviter le "VLAN Hopping".

B. Spanning Tree Protocol (STP)

Si vous bouclez deux câbles entre deux switchs pour la redondance, vous créez une Tempête de Broadcast (le réseau tombe en 2 secondes).
Rôle du STP : Détecter la boucle et bloquer logiquement un port.

  • Root Bridge : Le chef du réseau L2 (celui qui a la plus petite priorité ou adresse MAC).
  • BPDU : Les paquets de "bonjour" que s'envoient les switchs pour surveiller la topologie.

3. Routage & Interconnexion

A. Router-on-a-Stick (Inter-VLAN)

Les VLANs sont isolés. Pour qu'ils se parlent, il faut un routeur. On utilise une seule interface physique découpée en sous-interfaces logiques.

Config Cisco - RoaS
Router(config)# interface g0/0.10 Router(config-subif)# encapsulation dot1Q 10 Router(config-subif)# ip address 192.168.10.254 255.255.255.0

B. Routage Dynamique OSPF

Plutôt que d'écrire 50 routes statiques, OSPF apprend le réseau tout seul.
Concepts Clés :

  • Link-State : Chaque routeur connait la carte complète du réseau.
  • Coût : OSPF choisit le chemin le plus RAPIDE (Bande passante), pas le plus court (comme RIP).
  • Area 0 (Backbone) : Obligatoire. Toutes les autres aires doivent y être connectées.
  • Wildcard Mask : L'inverse du masque de sous-réseau (ex: /24 = 0.0.0.255).

4. Transport : TCP vs UDP

A. Le "Three-Way Handshake" TCP

Avant d'envoyer des données, TCP vérifie que l'autre est prêt. C'est une connexion FIABLE.

  1. SYN : "Bonjour, je veux te parler."
  2. SYN-ACK : "Bonjour, j'ai bien reçu, je suis d'accord."
  3. ACK : "Parfait, j'envoie les données."

B. Comparatif Express

  • TCP : Lent mais sûr. Réordonne les paquets, renvoie les perdus (Web, Mail, Transfert Fichier).
  • UDP : Rapide mais "bête". Si un paquet est perdu, tant pis (Streaming, VoIP, DNS, Jeux en ligne).

5. Sécurité : ACL & NAT

A. ACL (Access Control Lists)

Le pare-feu du routeur. Lit les règles de haut en bas.

  • Standard (1-99) : Filtre uniquement sur l'IP SOURCE. (À placer le plus près possible de la destination).
  • Etendue (100-199) : Filtre sur Source, Destination, Protocole et Port. (À placer le plus près possible de la source).

Implicit Deny : Il y a toujours un "Deny All" invisible à la fin d'une ACL. Si vous n'autorisez rien explicitement, tout est bloqué.

B. NAT (Network Address Translation)

  • NAT Dynamique (PAT) : "Many-to-One". Tout le bureau sort sur Internet avec l'unique IP Publique de la Box. Utilise les ports pour différencier les PC.
  • NAT Statique : "One-to-One". Pour publier un serveur Web (DMZ) accessible depuis l'extérieur.
